# 게임 디자인 전문가 철학 참고 문서

## 1. Jesse Schell — 구조 분석

**핵심 저서**: "The Art of Game Design: A Book of Lenses" (2008)

### Elemental Tetrad
게임을 4개의 기본 요소로 분해한다:
- **Mechanics**: 규칙과 절차. 게임이 어떻게 작동하는가
- **Story**: 게임 내 사건의 순서와 결과
- **Aesthetics**: 감각적 경험 (시각, 청각, 촉각)
- **Technology**: 게임을 구현하는 매체와 도구

**핵심 원칙**: 4축은 상호 지지적이어야 한다. 메커닉이 스토리를 구현하고, 미학이 감정을 강화하며, 기술이 이 모든 것을 가능케 해야 한다.

### 주요 렌즈
- **Essential Experience Lens**: 게임의 본질적 경험은 무엇인가? 이것이 모든 설계 결정의 기준이 된다
- **Emergence Lens**: 단순한 규칙들이 복잡하고 예상치 못한 행동을 만들어내는가
- **Interest Curve Lens**: 경험의 강도가 시간에 따라 어떻게 변하는가 (클라이맥스와 휴식의 리듬)
- **Flow Lens**: 플레이어가 몰입 상태를 유지하는가 (Csikszentmihalyi의 Flow 이론 적용)
- **Theme Lens**: 게임의 테마와 메커닉이 공명하는가

## 2. Raph Koster — 재미의 이론

**핵심 저서**: "A Theory of Fun for Game Design" (2004)

### 핵심 명제
> "재미란 패턴을 학습하는 과정에서 오는 쾌감이다."

게임은 본질적으로 **교육 기계**다. 뇌가 패턴을 인식하고 마스터할 때 도파민이 분비된다. 패턴이 완전히 마스터되면 게임은 지루해진다.

### 분석 프레임워크
- **핵심 패턴 식별**: 플레이어가 실제로 학습하는 것은 무엇인가
- **패턴 깊이 (Pattern Depth)**: 얼마나 오래 플레이어를 바쁘게 만드는가
- **Grokking Point**: 패턴을 완전히 이해한 이후 흥미를 유지하는 메커니즘
- **Chunking**: 복잡한 시스템을 인지적으로 감당 가능한 단위로 분해
- **Emergent Complexity**: 단순 규칙에서 나오는 복잡성의 양

## 3. MDA 프레임워크

**출처**: Hunicke, LeBlanc, Zubek, "MDA: A Formal Approach to Game Design" (GDC 2001)

### 3계층 모델
```
디자이너 관점: Mechanics → Dynamics → Aesthetics
플레이어 관점: Aesthetics ← Dynamics ← Mechanics
```

### 8가지 Aesthetics (미학 유형)
1. **Sensation** - 감각적 쾌감 (시각, 청각, 촉각)
2. **Fantasy** - 현실 도피, 역할극
3. **Narrative** - 드라마, 스토리
4. **Challenge** - 장애물 극복, 마스터리
5. **Fellowship** - 협력, 커뮤니티
6. **Discovery** - 탐험, 새로운 것을 찾는 기쁨
7. **Expression** - 자기 표현, 창의성
8. **Submission** - 휴식, 오락으로서의 파스타임

### 핵심 원칙
- 디자이너는 M에서 A로 설계하지만, 플레이어는 A에서 M을 경험한다
- 의도한 Aesthetic과 실제 경험 사이의 간극(Gap)이 핵심 개선 영역

## 4. Tynan Sylvester — 감정 공학

**핵심 저서**: "Designing Games: A Guide to Engineering Experiences" (2013)

### 핵심 명제
> "게임은 경험을 생성하는 기계다. 경험은 감정의 흐름이다."

### 감정 체인
```
Mechanics → Fiction → Events → Emotions → Experience
```

### 10가지 감정 트리거
1. **Learning** - 새로운 것을 배우는 만족감
2. **Character Arc** - 캐릭터의 변화와 성장
3. **Challenge** - 어려움을 극복하는 성취감
4. **Social** - 다른 사람과의 연결
5. **Acquisition** - 수집, 소유의 쾌감
6. **Music** - 음악이 유발하는 감정
7. **Spectacle** - 시각적 충격
8. **Beauty** - 미적 감동
9. **Environment** - 공간과 분위기
10. **Primal Threats** - 생존 본능 자극

### 미귀인 효과 (Misattribution)
메커닉이 생리적 각성을 만들고, 픽션이 그 각성에 라벨을 붙인다.
예: 심박수 증가(메커닉) + "적이 나타났다"(픽션) = 공포/흥분

### 엘레강스
메커닉의 수 대비 의미 있는 전략/경험의 비율이 높을수록 우아한 설계

## 5. Shigeru Miyamoto — 접근성 설계

**핵심 작품**: 마리오, 젤다, 동키콩

### 핵심 원칙
> "환경이 가르쳐야 한다. 매뉴얼 없이도 플레이할 수 있어야 한다."
> "게임은 보편적으로 접근 가능해야 한다."

### 설계 철학
- **환경 교육 (Environmental Teaching)**: 튜토리얼 텍스트 없이 레벨 배치가 직접 가르침
- **즉각적 반응성**: 입력에 대한 즉각적이고 명확한 피드백
- **A-ha! 모먼트**: 플레이어가 스스로 발견하게 설계된 "유레카" 순간
- **넓은 입구, 좁은 출구**: 쉽게 시작하고, 마스터리는 선택
- **기술-경험 통합**: 플랫폼의 고유한 특성을 활용한 경험 설계

### 온보딩 곡선
1~3분 내에 핵심 루프를 경험하게 하라. 이탈 포인트를 최소화하라.

## 6. Hidetaka Miyazaki — 극복의 미학

**핵심 작품**: 데몬즈 소울즈, 다크 소울 시리즈, 블러드본, 엘든 링

### 핵심 명제
> "세계 자체가 이야기를 말해야 한다."
> "높은 난이도는 보상의 전제 조건이다."

### 설계 철학
- **환경 스토리텔링**: 텍스트가 아닌 세계의 시각적 배치로 서사 전달
- **모호함의 가치**: 모든 것을 설명하지 않음으로써 플레이어의 상상과 해석을 유도
- **고난이도 + 공정성**: 어렵지만 패턴을 학습하면 반드시 클리어 가능한 설계
- **극복의 보상**: 어려울수록 클리어 시 만족감이 크다
- **발견의 기쁨**: 숨겨진 것을 찾는 탐험의 보상

### 감정 아크
좌절 → 반복 학습 → 패턴 파악 → 극복 → 강한 성취감 → 다음 도전 욕구

## 통합 교차 분석 가이드

6개 관점을 통합할 때 확인할 질문들:

1. **메커닉-스토리 공명**: Schell의 테트라드 관점에서 메커닉과 스토리가 서로를 지지하는가
2. **패턴 vs 감정**: Koster의 패턴 학습 관점과 Sylvester의 감정 트리거가 어떻게 연결되는가
3. **의도-경험 간극**: MDA 관점에서 디자이너 의도와 플레이어 실제 경험의 차이는 어디서 발생하는가
4. **접근성 vs 깊이**: Miyamoto의 접근성과 Miyazaki의 극복 미학 사이의 균형은 어떻게 잡혀 있는가
5. **혁신 포인트**: 어느 전문가도 예측하지 못한, 게임만의 독창적인 설계 요소는 무엇인가
